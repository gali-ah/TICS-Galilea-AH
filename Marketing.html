Marketing

Implementa estrategias para dar a conocer la cafetería, atraer más clientes y destacar la marca usando campañas, redes sociales y promociones en medios físicos y digitales.

Usuarios del modulo

  • Propietario

Requerimientos por usuario

Requerimientos del propietario

  • Registrar estrategias (plataformas como Facebook, Instagram, TikTok, etc.).
  • Registrar publicaciones con su título, contenido y fecha.
  • Registrar estadísticas de cada post (likes, vistas y compartidos).
  • Visualizar los datos en una tabla con la información de posts y estadísticas.
  • Filtrar la información por plataforma o por fecha.
  • Mantener la información disponible para consultas posteriores.

Diseño de la base de datos del módulo

Descripción de Pertinencia de la Arquitectura de la Base de Datos

Tabla Estrategia

Propósito

Registrar las plataformas o estrategias de marketing (Facebook, Instagram, TikTok, etc.) donde se publican campañas o posts.

Campos

  • Id_estrategia (INT, PK, AUTO_INCREMENT): Identificador único de cada estrategia/plataforma.
  • Nombre (NVARCHAR(100), NOT NULL): Nombre de la estrategia o red social (ej. "Facebook Ads").
  • Descripcion (NVARCHAR(200), NULL): Descripción breve de la estrategia (ej. "Campañas pagadas en Facebook").

Tabla Post

Propósito

Almacenar los posts o campañas publicadas en una estrategia específica.

Campos

  • Id_post (INT, PK, AUTO_INCREMENT): Identificador único del post.
  • Id_estrategia (INT, FK): Relación con la tabla Estrategia, indica en qué plataforma se publicó.
  • Titulo (NVARCHAR(150), NOT NULL): Título breve del post.
  • Contenido (NVARCHAR(500), NULL): Texto descriptivo o detalle del post.
  • Fecha_publicacion (DATE, NOT NULL): Fecha en que se publicó el post.

Tabla EstadisticasPost

Propósito

Registrar el rendimiento de los posts a lo largo del tiempo (likes, vistas, compartidos).

Campos

  • Id_estadistica (INT, PK, AUTO_INCREMENT): Identificador único de cada registro de estadísticas.
  • Id_post (INT, FK): Relación con la tabla Post, indica a qué publicación corresponden las métricas.
  • Likes (INT, DEFAULT 0): Número de "me gusta" obtenidos.
  • Vistas (INT, DEFAULT 0): Número de visualizaciones.
  • Compartidos (INT, DEFAULT 0): Número de veces que fue compartido.
  • Fecha_registro (DATE, DEFAULT GETDATE()): Fecha en que se registraron estas estadísticas.

Normalización de la Base de Datos

  • 1FN (Primera Forma Normal)
    • No hay atributos multivaluados ni repetidos.
    • Cada campo almacena un valor atómico.
  • 2FN (Segunda Forma Normal)
    • Todas las tablas tienen clave primaria.
    • Los campos dependen únicamente de la clave primaria:
      • En Estrategia, los campos dependen de Id_estrategia.
      • En Post, los campos dependen de Id_post.
      • En EstadisticasPost, los campos dependen de Id_estadistica.
  • 3FN (Tercera Forma Normal)
    • No existen dependencias transitivas.
    • Ejemplo: los Likes, Vistas y Compartidos dependen solo de la estadística, no de la estrategia.
    • Los nombres de plataformas están en su propia tabla (Estrategia), evitando redundancia.

Código SQL para crear la tabla

CREATE DATABASE ModuloMarketing;
go


CREATE TABLE Estrategia (
    Id_estrategia INT PRIMARY KEY IDENTITY(1,1),
    Nombre NVARCHAR(100) NOT NULL,  -- Ej. Facebook, Instagram
    Descripcion NVARCHAR(200) NULL
);


CREATE TABLE Post (
    Id_post INT PRIMARY KEY IDENTITY(1,1),
    Id_estrategia INT NOT NULL,
    Titulo NVARCHAR(150) NOT NULL,
    Contenido NVARCHAR(500) NULL,
    Fecha_publicacion DATE NOT NULL,
    FOREIGN KEY (Id_estrategia) REFERENCES Estrategia(Id_estrategia)
);


CREATE TABLE EstadisticasPost (
    Id_estadistica INT PRIMARY KEY IDENTITY(1,1),
    Id_post INT NOT NULL,
    Likes INT DEFAULT 0,
    Vistas INT DEFAULT 0,
    Compartidos INT DEFAULT 0,
    Fecha_registro DATE NOT NULL DEFAULT GETDATE(),
    FOREIGN KEY (Id_post) REFERENCES Post(Id_post)
);
