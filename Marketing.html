Marketing

Implementa estrategias para dar a conocer la cafetería, atraer más clientes y destacar la marca usando campañas, redes sociales y promociones en medios físicos y digitales.

Usuarios del modulo

  • Propietario

Requerimientos por usuario

Requerimientos del propietario

  • Crear Promociones: Definir nuevas campañas de marketing.
  • Acceder a Campañas Existentes: Consultar información sobre promociones activas.
  • Modificar Detalles de Campañas: Actualizar nombres, objetivos y medios de promoción.
  • Gestionar Fechas: Establecer y modificar fechas de inicio y fin de las promociones.
  • Eliminar Campañas Vencidas: Retirar promociones que ya no están en vigencia.

Diseño de la base de datos del módulo

Descripción de Pertinencia de la Arquitectura de la Base de Datos

Tabla Marketing

Propósito

La tabla Marketing tiene como objetivo gestionar las campañas de marketing de la cafetería. Permite crear, consultar, actualizar y eliminar campañas para medir su impacto en clientes y ventas.

Campos

  • Id_marketing: Identificador único de la campaña (clave primaria, autoincrementable).
  • Nombre: Nombre de la campaña (ejemplo: Promo 2x1 Café).
  • Objetivo: Propósito de la campaña (ejemplo: Aumentar clientes nuevos).
  • Medio: Canal de difusión (ejemplo: Facebook, radio, volanteo, etc.).
  • Fecha_inicio: Fecha en que inicia la campaña.
  • Fecha_fin: Fecha en que finaliza la campaña (puede estar vacía si sigue activa).

Normalización de la Base de Datos

  • 1FN: Cada campo almacena un único valor; la tabla Marketing cumple.
  • 2FN: Todos los campos dependen de la clave primaria (Id_marketing).
  • 3FN: Ningún campo depende de otro campo que no sea la clave primaria.

Código SQL para crear la tabla

-- Crear la base de datos
CREATE DATABASE MonkyCafeDB;
GO

-- Usar la base de datos
USE MonkyCafeDB;
GO

-- Crear tabla Marketing
CREATE TABLE Marketing (
    Id_marketing INT PRIMARY KEY IDENTITY(1,1),
    Nombre NVARCHAR(100) NOT NULL,
    Objetivo NVARCHAR(200),
    Medio NVARCHAR(100),
    Fecha_inicio DATE NOT NULL,
    Fecha_fin DATE
);
GO
