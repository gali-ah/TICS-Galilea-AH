Carta

Crear y organizar un menú atractivo y fácil de entender que ayude a los clientes a elegir productos y aumente las ventas, disponible en formato físico y digital.

Usuarios del modulo

  • Cliente
  • Propietario

Requerimientos por usuario

Requerimientos del Cliente

  • Consultar el menú completo de manera clara y organizada.
  • Visualizar los productos por categoría y subcategoría.
  • Identificar qué productos están disponibles o no.
  • Ver las promociones activas con precio original y precio con descuento.
  • Contar con información básica del producto: nombre, descripción, ingredientes principales, precio.

Requerimientos del Propietario

  • Poder registrar y actualizar categorías, subcategorías y productos.
  • Controlar la disponibilidad de los platillos (activar o desactivar según stock).
  • Gestionar fácilmente las promociones para destacar descuentos.
  • Mantener la base de datos en un formato normalizado que evite redundancias.

Diseño de la base de datos del módulo

Descripción de Pertinencia de la Arquitectura de la Base de Datos

Tabla Categorias

Propósito

Agrupar los productos en grandes categorías del menú (ejemplo: Hamburguesas, Bebidas, Postres, Baguettes).

Campos

  • id_categoria: Identificador único (PK).
  • nombre: Nombre de la categoría (ej. "Bebidas").
  • descripcion: Breve descripción de la categoría.

Tabla Subcategorias

Propósito

Dividir cada categoría en subgrupos más específicos.

Campos

  • id_subcategoria: Identificador único (PK).
  • id_categoria: Llave foránea que indica a qué categoría pertenece.
  • nombre: Nombre de la subcategoría (ej. "Hamburguesas Clásicas").
  • descripcion: Breve detalle de la subcategoría.

Tabla Productos

Propósito

Almacenar la información de cada producto del menú (platillos, bebidas, postres, etc.).

Campos

  • id_producto: Identificador único (PK).
  • id_subcategoria: Llave foránea que conecta el producto con su subcategoría.
  • nombre: Nombre del producto.
  • descripcion: Texto con detalle breve del producto.
  • ingredientes: Ingredientes principales.
  • precio: Precio normal del producto.
  • precio_promocion: Precio con descuento (si aplica).
  • disponible: BIT → 1 = disponible, 0 = no disponible.
  • tamaño: Aplica solo para bebidas.
  • sabor: Aplica en postres/helados.

Normalización de la Base de Datos

  • FN (Primera Forma Normal)
    • Cada campo almacena un valor atómico (sin listas ni valores repetidos).
    • Ejemplo: ingredientes se manejan como texto, no como múltiples columnas.
  • 2FN (Segunda Forma Normal)
    • Todas las tablas dependen de su clave primaria.
    • Ejemplo: en Subcategorias, el campo nombre depende de id_subcategoria, no de otra cosa.
  • 3FN (Tercera Forma Normal)
    • No existen dependencias transitivas.
    • Ejemplo: un producto no guarda el nombre de la categoría directamente, solo el id_subcategoria, y desde ahí se obtiene la categoría → evita duplicar datos.

Código SQL para crear la tabla

CREATE TABLE Categorias (
    id_categoria INT PRIMARY KEY IDENTITY(1,1),
    nombre NVARCHAR(100) NOT NULL,   -- Hamburguesas, Bebidas, Postres, Baguettes
    descripcion NVARCHAR(255)
);

-- TABLA DE SUBCATEGORÍAS
CREATE TABLE Subcategorias (
    id_subcategoria INT PRIMARY KEY IDENTITY(1,1),
    id_categoria INT NOT NULL,
    nombre NVARCHAR(100) NOT NULL,  -- Ejemplo: Hamburguesas Clásicas, Refrescos, Helados
    descripcion NVARCHAR(255),
    FOREIGN KEY (id_categoria) REFERENCES Categorias(id_categoria)
);

-- TABLA DE PRODUCTOS
CREATE TABLE Productos (
    id_producto INT PRIMARY KEY IDENTITY(1,1),
    id_subcategoria INT NOT NULL,
    nombre NVARCHAR(150) NOT NULL,         -- Nombre del platillo
    descripcion NVARCHAR(500),             -- Descripción corta
    ingredientes NVARCHAR(500) NULL,       -- Ingredientes (NULL en bebidas)
    precio DECIMAL(10,2) NOT NULL,         -- Precio base
    precio_promocion DECIMAL(10,2) NULL,   -- Precio con descuento
    disponible BIT DEFAULT 1,              -- TRUE = disponible / FALSE = no disponible
    tamanio NVARCHAR(50) NULL,             -- Solo aplica en bebidas
    sabor NVARCHAR(50) NULL,               -- Solo aplica en postres/helados
    FOREIGN KEY (id_subcategoria) REFERENCES Subcategorias(id_subcategoria)
);
